Este documento cubre los pasos necesarios para realizar copias de seguridad y recuperaciones de Microsoft SQL Server en AWS FSx ONTAP con SnapCenter.  Esto incluye la siguiente información:

* Configuración de NetApp SnapCenter
* Operaciones de copia de seguridad de SnapCenter
* Operación de respaldo para una base de datos de FCI
* Operación de respaldo para múltiples bases de datos
* Restaurar y recuperar




== Configuración de SnapCenter

Se deben realizar los siguientes pasos para la configuración de SnapCenter y la protección de los recursos de Microsoft SQL Server.  Cada uno de los siguientes pasos se detalla en las siguientes secciones.

. Configure las credenciales de administrador del sistema para el usuario de copia de seguridad y restauración de SQL Server.
. Configurar los ajustes de almacenamiento.  Proporcione las credenciales de administración de Amazon Web Services (AWS) para acceder a las máquinas virtuales de almacenamiento (SVM) de Amazon FSx ONTAP desde SnapCenter.
. Agregue un host de SQL Server a SnapCenter.  Implemente e instale los complementos de SnapCenter necesarios.
. Configurar políticas.  Defina el tipo de operación de respaldo, la retención y la replicación de respaldo de instantánea opcional.
. Configurar y proteger la base de datos de Microsoft SQL Server.




== Interfaz de usuario recién instalada de SnapCenter

Configurar credenciales para la copia de seguridad de SQL Server y restaurar al usuario con derechos de administrador del sistema.

image:sqlsb-aws-001.png["Figura que muestra el diálogo de entrada/salida o representa contenido escrito"]

NetApp recomienda utilizar el control de acceso basado en roles (RBAC) para delegar capacidades de gestión y protección de datos a usuarios individuales en los hosts de SnapCenter y Windows.  El usuario debe tener acceso al servidor SQL que aloja la base de datos.  Para varios hosts, el nombre de usuario y la contraseña deben ser los mismos en todos los hosts.  Además, para permitir que SnapCenter implemente el complemento requerido en los hosts de SQL Server, debe registrar la información del dominio para que SnapCenter valide sus credenciales y hosts.

Amplíe las siguientes secciones para ver las instrucciones detalladas sobre cómo completar cada paso.

== Configurar una política de copias de seguridad

Una política de respaldo es un conjunto de reglas que rigen cómo administrar, programar y conservar las copias de seguridad.  Ayuda con el tipo y la frecuencia de las copias de seguridad en función del SLA de su empresa.

Amplíe las siguientes secciones para ver las instrucciones detalladas sobre cómo completar cada paso.

.Configurar la operación de respaldo para una base de datos FCI
[%collapsible%open]
====
Para configurar una política de respaldo para una base de datos FCI, complete los siguientes pasos:

. Vaya a *Configuración* y seleccione *Políticas* en la parte superior izquierda.  Luego haga clic en *Nuevo*.
+
image:sqlsb-aws-016.png["Figura que muestra el diálogo de entrada/salida o representa contenido escrito"]

. Introduzca el nombre de la política y una descripción.  Haga clic en *Siguiente*.
+
image:sqlsb-aws-017.png["Figura que muestra el diálogo de entrada/salida o representa contenido escrito"]

. Seleccione *Copia de seguridad completa* como tipo de copia de seguridad.
+
image:sqlsb-aws-018.png["Figura que muestra el diálogo de entrada/salida o representa contenido escrito"]

. Seleccione la frecuencia de programación (esto se basa en el SLA de la empresa).  Haga clic en *Siguiente*.
+
image:sqlsb-aws-019.png["Figura que muestra el diálogo de entrada/salida o representa contenido escrito"]

. Configure los ajustes de retención para la copia de seguridad.
+
image:sqlsb-aws-020.png["Figura que muestra el diálogo de entrada/salida o representa contenido escrito"]

. Configurar las opciones de replicación.
+
image:sqlsb-aws-021.png["Figura que muestra el diálogo de entrada/salida o representa contenido escrito"]

. Especifique un script de ejecución para ejecutar antes y después de ejecutar un trabajo de copia de seguridad (si corresponde).
+
image:sqlsb-aws-022.png["Figura que muestra el diálogo de entrada/salida o representa contenido escrito"]

. Ejecute la verificación según el programa de copia de seguridad.
+
image:sqlsb-aws-023.png["Figura que muestra el diálogo de entrada/salida o representa contenido escrito"]

. La página *Resumen* proporciona detalles de la política de respaldo.  Cualquier error se puede corregir aquí.
+
image:sqlsb-aws-024.png["Figura que muestra el diálogo de entrada/salida o representa contenido escrito"]

== Restaurar y recuperar

Consulte los siguientes requisitos previos necesarios para restaurar una base de datos de SQL Server en SnapCenter.

* La instancia de destino debe estar en línea y ejecutándose antes de que se complete un trabajo de restauración.
* Las operaciones de SnapCenter que están programadas para ejecutarse en la base de datos de SQL Server deben estar deshabilitadas, incluidos todos los trabajos programados en servidores de administración remota o de verificación remota.
* Si está restaurando copias de seguridad del directorio de registro personalizado en un host alternativo, el servidor de SnapCenter y el host del complemento deben tener la misma versión de SnapCenter instalada.
* Puede restaurar la base de datos del sistema en un host alternativo.
* SnapCenter puede restaurar una base de datos en un clúster de Windows sin desconectar el grupo de clústeres de SQL Server.
